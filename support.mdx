## Common Support Topics

### 1. API Keys

<AccordionGroup>
  <Accordion title="Lost API Key">
    For security reasons, we cannot recover lost API keys. Instead: 1. Log in to
    your account 2. Generate a new API key 3. Update your applications 4. Delete
    the old key
  </Accordion>
  <Accordion title="Rate Limits">
    Default limits: - 1000 requests per hour for most endpoints - 100 order
    creations per hour - 50 order confirmations per hour Need higher limits?
    Email support with: - Your use case - Expected volume - Current limitations
  </Accordion>
  <Accordion title="Key Rotation">
    Best practices for key rotation: 1. Generate new key 2. Update applications
    gradually 3. Monitor for errors 4. Delete old key after confirming
    everything works
  </Accordion>
</AccordionGroup>

### 2. Orders & Payments

<AccordionGroup>
  <Accordion title="Failed Orders">
    If an order fails: 1. Check the error message 2. Verify input data 3. Ensure
    sufficient funds 4. Contact support if issue persists
  </Accordion>
  <Accordion title="Payment Identifiers">
    For IMAD/OMAD issues: 1. Verify order status 2. Check payment details 3.
    Allow processing time 4. Contact support for urgent requests
  </Accordion>
  <Accordion title="Order Status">
    Order status flow: 1. `pending`: Initial state 2. `submitted`: Order
    confirmed 3. `raised`: Funds detected 4. `sent`: Transfer initiated 5.
    `received`: Transfer completed
  </Accordion>
</AccordionGroup>

### 3. Integration Help

## Before Contacting Support

1. Check the documentation
2. Search for similar issues
3. Gather relevant information:
   - API key (last 4 characters)
   - Order/Recipient IDs
   - Error messages
   - Steps to reproduce
   - Timestamps

## Security Issues

For security concerns:

1. Email: security@example.com
2. Include "SECURITY" in subject
3. Encrypt sensitive data
4. Expect response within 1 hour

<Note>
  Never share your full API key or sensitive customer data when contacting
  support.
</Note>
