This guide explains how to create, manage, and secure your API keys.

## Creating API Keys

1. Log in to your SD2 Mod account
2. Navigate to **Settings** > **API Keys**
3. Click **Generate New Key**
4. Save your key securely - it won't be shown again

<Warning>
  Never share your API key or commit it to version control. Always use
  environment variables to store sensitive credentials.
</Warning>

## Best Practices

### 1. Secure Storage

Store your API keys using environment variables:

<CodeGroup>

```typescript TypeScript
// Load from .env file
import 'dotenv/config'

const apiKey = process.env.API_KEY
if (!apiKey) {
  throw new Error('API key not configured')
}
```

```python Python
import os
from dotenv import load_dotenv

# Load from .env file
load_dotenv()

api_key = os.getenv('API_KEY')
if not api_key:
    raise ValueError('API key not configured')
```

</CodeGroup>

### 2. Key Rotation

- Generate new keys before old ones expire
- Update applications gradually
- Monitor for errors during transition
- Delete old keys after confirming everything works

### 3. Access Control

- Use separate keys for different environments
- Limit key access to necessary services
- Revoke compromised keys immediately

## Rate Limits

- Development keys: 1000 requests per hour
- Production keys: 10000 requests per hour
- Contact support for custom limits
